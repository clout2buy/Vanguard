# Implement dependency-aware plugin lifecycle

Complete the plugin system under `src/` while preserving its public exports. `PluginRegistry.register(plugin)` accepts plugins with a unique non-empty `name`, optional unique `requires` names, and async `start()` / `stop()` functions. Reject malformed and duplicate registrations. `startAll()` must validate missing dependencies and cycles before starting anything, then start plugins once in stable dependency order (registration order breaks ties). If any start fails, stop every plugin successfully started during that call in reverse order and rethrow the original failure. `stopAll()` stops all currently started plugins in reverse actual start order, attempts every stop even when one fails, clears lifecycle state, and throws an aggregate error after cleanup when needed. Repeated `startAll` and `stopAll` calls must be safe. `status()` returns a new object mapping registered names to `"registered"` or `"started"`. Do not expose mutable internal state.

Use `check_project` for the trusted public behavior suite before final review.
